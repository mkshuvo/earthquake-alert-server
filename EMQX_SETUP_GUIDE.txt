Certificate-Based Authentication (X.509)
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  For IoT sensors, use certificate-based authentication:
  
  1. Set in emqx.conf:
     listeners.ssl.default {
       ssl_opts {
         verify = required          # Require client certificate
         fail_if_no_peer_cert = true
       }
     }
  
  2. Generate sensor certificate with specific CN:
     openssl req -new ... -subj "/CN=earthquake-sensor"
  
  3. Connect with certificate:
     mosquitto_pub -h localhost -p 47754 \
       --cert client-cert.pem \
       --key client-key.pem \
       --cafile ca-cert.pem \
       -t "earthquake/sensors/001/data" \
       -m '{"magnitude": 5.2}'

  Currently configured in emqx.conf:

    discovery_strategy = static
    static {
      seeds = ["emqx@emqx"]
    }
  }

  To add nodes:
    1. Update docker-compose.yml to add new services (emqx2, emqx3, etc.)
    2. Update seeds in emqx.conf:
       seeds = ["emqx@emqx", "emqx@emqx2", "emqx@emqx3"]
    3. Ensure all nodes can reach each other
    4. Restart: docker-compose restart

    2. For production, use wildcard or SAN certificates
    3. Update ca-cert.pem if using different CA
